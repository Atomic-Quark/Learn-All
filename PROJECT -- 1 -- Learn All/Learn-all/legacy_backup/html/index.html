HTML Mastery

The standard markup language for creating web pages.

What is HTML? (The Skeleton)

Imagine a human body. It needs a skeleton to hold it up, right? Without bones, we'd just be a pile of jelly!

HTML is the skeleton of a website.

  • It tells the browser "This is a heading".
  • It says "This is a paragraph".
  • It says "Put an image here".

It doesn't make things look pretty (that's CSS's job, the "skin" and "clothes"), but it gives everything structure.

Your First Code

HTML uses "tags". Tags are like little containers. They look like this: <tagname>.

<h1>Hello World!</h1>
// This makes a big heading

<p>I am learning to code.</p>
// This makes a paragraph of text

Essential Tags

<h1> to <h6>

Headings. h1 is the biggest, h6 is the smallest.

<p>

Paragraph. Used for normal text.

<a href="...">

Anchor (Link). Connects to other pages.

<img src="...">

Image. Shows a picture.

The HTML Structure

Every HTML file needs a standard setup. Think of it like a sandwich.

<!DOCTYPE html> // Tells browser "This is HTML5"
<html> // The top bun
  <head> // Brain (settings, title)
    <title>My Page</title>
  </head>
  <body> // The meat (visible content)
    <h1>Welcome!</h1>
  </body>
</html> // The bottom bun

Lists & Tables

Organizing data is key. We use lists for items and tables for grids.

Lists

<ul> // Unordered List (Bullets)
  <li>Milk</li>
  <li>Eggs</li>
</ul>

Tables

<table>
  <tr> // Table Row
    <th>Name</th> // Header
    <td>John</td> // Data
  </tr>
</table>

Forms & Input

Forms allow users to send data to the server. This is how logins and searches work.

<form>
  <!-- Text Input -->
  <input type="text" placeholder="Username">

  <!-- Password -->
  <input type="password" placeholder="Secret">

  <!-- Button -->
  <button>Login</button>
</form>

Semantic HTML

Using the right tag for the right job. It helps search engines (SEO) and screen readers.

Don't use <div> for everything!
Use <header> for tops.
Use <nav> for links.
Use <footer> for bottoms.

HTML5 APIs

Modern HTML can do amazing things without plugins.

1. Geolocation

Get the user's position (with permission).

2. LocalStorage

Save data in the browser (like we do for this app!).

3. Canvas

Draw graphics dynamically.

<canvas id="myGame" width="200" height="100"></canvas>
